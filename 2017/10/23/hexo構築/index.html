hexo構築

Posted on 2017-10-23

構築する前、Node.jsとGitという環境が必要。
Node.jsだったら、ホームページでダンロードできる。
GitにはXcodeをインストールしたらついてるから、Githubの申請から話す。
GithubはBlogのリポジトリ(Repository)やサーバーの役割のためで、普通にGithubのホームページでアカウントを作成したらオーケー。

ここからは本番！

NodeとGitをインストールできたら、まず空フォルダを作る、名前は一応blogにしたらわかりやすい。

mkdir blog

Hexoをインストールするの実行コマンド：

sudo npm install -g hexo

そして、gitの時と同じく、初期化する：

hexo init

これでインストールを全部終わったw
中には確か：

_config.yml package.json scaffolds themes

などのファイルとかがある

これからのページは全部blogフォルダーの中に作る

静的ページを作るコマンド：

hexo generate

ローカルで起動する：

hexo server

http://localhost:4000でチェックできる(パソコンの環境によって、できない人もいるだと思う、これは気にしなくってもいい、「ctrl + c」で止める)

###Githubを配置する

####Repositoryを構築
Repositoryの名前は必ず『Githubのユーザー名.github.io』という形で作る

そして、先作ったblogフォルダーに_config.ymlを修正する

vim _config.yml

一番下でこうように書く

deploy:

type: git

repo: https://github.com/Githubのユーザー名/Githubのユーザ名.github.io.git

branch: master

Ps: vimではIを押したら書ける、Escを押して「：」を押して、そして「wq」を入力したら、保存できる。

終わったら、次のコマンドを実行する：

npm install hexo-deployer-git --save

バージョンによって、このコマンドの書き方が変わるかもしれない

それから、実行する：

hexo deploy

ウェブブラウザーでhttp://Githubのユーザー名.github.io.gitを入力し、以下のような画像が出たら、おめでとう！できました！
￼

これ以降またの配置したいなら、だいたい三つのステップでできる

hexo clean

hexo generate

hexo deploy

よく使うコマンド 目的
hexo new page “pageName” 新しいページを作る
hexo generate publicディレクトリで静的ページを作る
hexo server ローカルアクセスを起動
hexo deploy ディレクトリをGitHubに配置する
hexo help ヘルップ
hexo version Hexoのバージョンチェック
hexo new “postName” 新しい文章を作る
# hexo # 总结
